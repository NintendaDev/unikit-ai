# Engine Rules: Godot 4

Engine-specific verification checks for Godot 4 (GDScript) projects.

## Companion File Checks

Godot does not use companion files like Unity's `.meta`. **Skip companion file checks.**

### .import file warning

If any `.import` files appear in `CHANGED_FILES`, emit a warning — these files are auto-generated by the Godot editor and should be in `.gitignore`:

```bash
echo "$CHANGED_FILES" | grep '\.import$'
```

If found: `WARN: .import files in changed files — these should be gitignored`

## Scene File Consistency (.tscn)

For each modified `.tscn` file in `CHANGED_FILES`:

1. **Script references** — verify that `[ext_resource]` entries pointing to `.gd` scripts reference files that exist
2. **Node structure** — check that `[node]` entries reference valid types (built-in or custom classes)
3. **Removed scripts** — if a `.gd` file was deleted, check that no `.tscn` still references it

```bash
# Find script references in changed .tscn files
grep -n 'path="res://.*\.gd"' <changed_tscn_files>
```

## project.godot Settings

If `project.godot` was modified in `CHANGED_FILES`:

1. **Autoload entries** — verify each autoload script/scene exists at the referenced path
2. **Main scene** — verify `run/main_scene` points to an existing `.tscn`
3. **Feature tags** — check `config/features` is consistent with the project's Godot version

## Autoload Validation

For each autoload defined in `project.godot` → `[autoload]` section:

- Verify the referenced file (`.gd` or `.tscn`) exists
- If it's a `.gd` file, verify it extends `Node` (or a Node subclass)

## addons/ Directory Structure

For modified files under `addons/`:

- Each addon subfolder should contain `plugin.cfg`
- Verify `plugin.cfg` has required fields: `name`, `description`, `version`
- Flag changes to third-party addons (addons not authored by the project)

## GDScript Type-Hint Conventions

For new or modified `.gd` files, check for type hints on:

- Function parameters: `func foo(bar: int)` — not `func foo(bar)`
- Function return types: `func foo() -> int:` — not `func foo():`
- Variable declarations: `var x: int = 0` — not `var x = 0`

Missing type hints are a warning (strict mode: failure).

## Module Boundary Checks

Godot has **no compile-time module boundary enforcement**. Skip automated boundary checks.

Defer to `.unikit/ARCHITECTURE.md` for convention-based boundary rules if available.

## Read-Only Paths

These directories must NOT be modified. Ignore them during checks and flag any changes as errors:

- `.godot/`
- `addons/` (third-party only — project-owned addons are fine)

## Strict Mode Items

Items that are **always checked** (both normal and strict mode) and always fail on violation:

| Check | Details |
|-------|---------|
| Scene file consistency | Script references in `.tscn` must point to existing files |
| GDScript type hints | Type annotations required on parameters, returns, and variables (strict mode only: failure) |
